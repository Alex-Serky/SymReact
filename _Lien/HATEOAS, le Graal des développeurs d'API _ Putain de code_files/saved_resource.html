        Super article ! En revanche je crois qu'il y a une petite inversion sur la partie POST et PUT.

        PUT est utilisé pour créer ou écraser une ressource en spécifiant la nouvelle URL. Par exemple, PUT /customers/123 va créer ou écraser le client numéro 123. Lorsqu'on utilise PUT, on s'attend à remplacer intégralement la ressource si elle existe déjà.

        POST peut être utilisé pour créer une ressource, mais sans spécifier un identifiant, comme POST /customers. Auquel cas on devrait renvoyer dans le résultat l'URL de la ressource créée pour pouvoir la manipuler ultérieurement.

        En revanche il est possible de mettre à jour partiellement une ressource existante avec POST /customers/123. On peut par exemple juste changer le nom du "customer" avec un POST.
